## Plan: Create `COCO_WORKSHOP.PIPELINE_LAB.SILVER_AP_INVOICES` Dynamic Table

### Context
- Two bronze AP invoice tables exist in `SOURCE_DATA` with equivalent semantics but different column names and value encodings.
- No dynamic tables currently exist in `PIPELINE_LAB`.
- The DDL below compiles successfully.

### Implementation (single DDL)

```sql
CREATE OR REPLACE DYNAMIC TABLE COCO_WORKSHOP.PIPELINE_LAB.SILVER_AP_INVOICES
  TARGET_LAG = '1 hour'
  WAREHOUSE = COCO_WORKSHOP_WH
AS
SELECT
    MD5(SOURCE_SYSTEM || '|' || INVOICE_ID) AS INVOICE_SK,
    SOURCE_SYSTEM,
    INVOICE_ID,
    INVOICE_NUMBER,
    VENDOR_ID,
    VENDOR_NAME,
    INVOICE_DATE,
    DUE_DATE,
    INVOICE_AMOUNT,
    CURRENCY_CODE,
    PAYMENT_TERMS,
    PO_NUMBER,
    LINE_DESCRIPTION,
    GL_ACCOUNT,
    COST_CENTER,
    APPROVAL_STATUS,
    CREATED_AT
FROM (
    -- SAP source
    SELECT
        'SAP' AS SOURCE_SYSTEM,
        INVOICE_ID, INVOICE_NUMBER, VENDOR_ID, VENDOR_NAME,
        INVOICE_DATE, DUE_DATE, INVOICE_AMOUNT, CURRENCY_CODE,
        PAYMENT_TERMS, PO_NUMBER, LINE_DESCRIPTION, GL_ACCOUNT,
        COST_CENTER, APPROVAL_STATUS, CREATED_AT
    FROM COCO_WORKSHOP.SOURCE_DATA.BRONZE_SAP_AP_INVOICES

    UNION ALL

    -- Oracle source (rename + normalize values)
    SELECT
        'ORACLE' AS SOURCE_SYSTEM,
        INV_ID, INV_NUM, SUPPLIER_ID, SUPPLIER_NAME,
        INV_DATE, PAYMENT_DUE_DATE, TOTAL_AMOUNT, CURRENCY,
        CASE TERMS_CODE WHEN 'N30' THEN 'NET30' WHEN 'N60' THEN 'NET60' ELSE TERMS_CODE END,
        PURCHASE_ORDER, DESCRIPTION, ACCOUNT_CODE,
        DEPT_CODE,
        CASE STATUS WHEN 'VALIDATED' THEN 'APPROVED' ELSE STATUS END,
        CREATION_DATE
    FROM COCO_WORKSHOP.SOURCE_DATA.BRONZE_ORACLE_AP_INVOICES
)
```

### Design decisions
| Decision | Choice | Rationale |
|----------|--------|-----------|
| `PAYMENT_TERMS` | Normalize Oracle `N30`/`N60` → `NET30`/`NET60` | Consistent with SAP format; easier downstream filtering |
| `APPROVAL_STATUS` | Map Oracle `VALIDATED` → `APPROVED` | Per earlier discussion these are functionally equivalent |
| `GL_ACCOUNT` / `COST_CENTER` | Pass through raw values (no cross-system mapping) | Flagged as open question; avoids hidden assumptions |
| Surrogate key | `MD5(SOURCE_SYSTEM \|\| '\|' \|\| INVOICE_ID)` | Guarantees uniqueness across sources |
| `TARGET_LAG` | 1 hour | Reasonable default for workshop; easily adjustable |

### Verification
- `SELECT COUNT(*) FROM COCO_WORKSHOP.PIPELINE_LAB.SILVER_AP_INVOICES` → expect 30 rows (15 SAP + 15 Oracle)
- `SELECT DISTINCT SOURCE_SYSTEM` → `SAP`, `ORACLE`
- `SELECT DISTINCT APPROVAL_STATUS` → `APPROVED`, `PENDING` (no `VALIDATED`)
- `SELECT DISTINCT PAYMENT_TERMS` → `NET30`, `NET60` (no `N30`/`N60`)
